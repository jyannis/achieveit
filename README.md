# commit规范
 type: description

type：

        feat:  新功能
        update:  更新
        refactor :  已有功能重构
        fix:  修复bug
        perf :  性能优化

        style :  代码格式改变
        test:  UT
        build:  构建工具或构建过程等的变动
        revert:  撤销上一次的commit
        docs :  文档

        merge :  纯粹分支合并

如: [feat]增加成本中心授权人联动。
